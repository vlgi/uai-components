# List

| CSS Property              | Descripiton              | Default   |
| ------------------------- | ------------------------ | --------- |
| --uai-list-display        | card list display        | flex      |
| --uai-list-flex-direction | card list flex direction | column    |
| --uai-list-padding        | card list padding        | 0 0.25rem |
| --uai-list-items-gap      | card list gap            | 0.5rem    |

This component use our Card component, so as css variables will cascade you can set the css
variables of the card for this component too. But as they cascade if you use our Card component
inside the list slot you must overwritte the css variables used to customize the list. Unfortunately
this is a limitation of the approach we choose to customize components styles.

```tsx
...
<List
  --card-background: red
>
  <svelte:fragment
    slot="list-item"
    let:text
  >
    <!--
      here we set the --card-background to blue even if it is the default
      because the var defined on list will cascate to here too
    -->
    <Card
      --card-background: blue
    >
      <span>{text}</span>
    </Card>
  </svelte:fragment>
</List>
...
```
